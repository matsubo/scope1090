# scope1090 Design Spec

**Date:** 2026-04-03
**Status:** Approved

---

## Overview

scope1090 is a complete rewrite of graphs1090 — an ADS-B receiver performance monitoring dashboard. It replaces the RRD/PNG pipeline with a SQLite time-series store and ECharts interactive visualizations, running on Raspberry Pi with minimal SD card writes.

**Key goals:**
- Store all metrics at full resolution (no data thinning for recent data)
- Interactive zoom/pan over arbitrary time ranges (Zabbix-style)
- Minimize SD card writes (RAM-first storage)
- Remove OS monitoring (CPU, disk, memory, network, temperature)
- Lightweight enough for Raspberry Pi

---

## Architecture

```
readsb/dump1090 (stats.json)
        ↓ every 60s
  collector.py ─────────────────────────────────────────┐
        ↓ INSERT                                         │ on boot: restore
  /run/scope1090/live.db  (tmpfs = RAM, WAL mode)        │
        ↓ hourly cp (systemd timer)                      │
  /var/lib/scope1090/archive.db  (SD card) ─────────────┘

  api.py (Flask)
        ← SELECT from live.db
        → GET /api/metrics
        → GET /api/metrics/names
        → GET /api/status

  nginx / lighttpd
        ← static files: /usr/share/scope1090/html/
        ← proxy /api/* → Flask :5000

  Browser (ECharts + Vite)
        ← polls /api/metrics every 60s
        → zoom/pan triggers re-query with new from/to
```

## Components

### collector.py

- Polls readsb `stats.json` every 60 seconds via `file://` or HTTP URL
- Writes rows to `/run/scope1090/live.db` (tmpfs)
- On startup: does nothing if live.db already exists (restored by boot service)
- Graceful shutdown: flushes any pending writes before exit

**Metrics collected (ADS-B only, no OS metrics):**

| Metric name | Source field | Unit |
|-------------|-------------|------|
| `aircraft` | `last1min.aircraft` | count |
| `messages_rate` | `last1min.local.accepted` / 60 | msg/s |
| `max_range` | `last1min.max_distance_m` / 1000 | km |
| `signal_mean` | `last1min.local.signal` | dBFS |
| `signal_peak` | `last1min.local.peak_signal` | dBFS |
| `tracks_total` | `total.tracks_with_position` | count |
| `airspy_rssi` | Airspy stats (if present) | dB |
| `airspy_snr` | Airspy stats (if present) | dB |

### db.py

SQLite helper module shared by collector and api.

**Schema:**

```sql
CREATE TABLE metrics (
    ts    INTEGER NOT NULL,
    name  TEXT    NOT NULL,
    value REAL    NOT NULL,
    PRIMARY KEY (ts, name)
) WITHOUT ROWID;

CREATE INDEX idx_name_ts ON metrics (name, ts);
```

**Data retention policy** (daily maintenance job):

| Age | Resolution |
|-----|-----------|
| 0 – 30 days | 1-minute (raw) |
| 30 days – 1 year | 1-hour average |
| > 1 year | deleted |

Maintenance SQL:
```sql
-- 1. Aggregate rows older than 30 days into hourly averages
INSERT OR REPLACE INTO metrics (ts, name, value)
    SELECT (ts / 3600) * 3600, name, AVG(value)
    FROM metrics
    WHERE ts < strftime('%s', 'now', '-30 days')
    GROUP BY (ts / 3600), name;

-- 2. Delete raw rows older than 30 days (keep only the hourly aggregates)
DELETE FROM metrics
    WHERE ts < strftime('%s', 'now', '-30 days')
      AND ts % 3600 != 0;

-- 3. Delete rows older than 1 year
DELETE FROM metrics
    WHERE ts < strftime('%s', 'now', '-1 year');
```

**live.db size estimate:**
- 8 metrics × 1 row/min × 60 min × 24 h × 30 days = 345,600 rows
- ~40 bytes/row → ~14 MB RAM

### api.py

Flask application serving JSON API. Reads from `/run/scope1090/live.db`.

**Endpoints:**

```
GET /api/metrics
    ?metric=aircraft     required
    &from=<unix_sec>     default: now - 86400
    &to=<unix_sec>       default: now
    &resolution=auto     auto | raw | 1h
    → { metric, from, to, resolution, data: [[ts, value], ...] }

GET /api/metrics/names
    → { names: ["aircraft", "messages_rate", ...] }

GET /api/status
    → { last_collected: <unix_sec>, db_size_bytes: <int>, uptime_sec: <int> }
```

**`resolution=auto` logic:**

| Query span | Resolution |
|-----------|-----------|
| ≤ 24 hours | raw (1 min) |
| > 24 hours | 1-hour average (GROUP BY hour) |

**Response format:**
```json
{
  "metric": "aircraft",
  "from": 1680000000,
  "to": 1680086400,
  "resolution": "1m",
  "data": [
    [1680000000, 42],
    [1680000060, 45]
  ]
}
```

### Frontend (Vite + ECharts)

**Tech stack:** Vite, ECharts 5, vanilla JS (no framework)

**Source structure:**
```
html/
├── src/
│   ├── main.js       # entry point, chart init, 60s polling loop
│   ├── charts.js     # ECharts option builders per metric
│   ├── api.js        # fetch wrappers for /api/* endpoints
│   └── controls.js   # time range selector, zoom event handling
├── index.html
├── package.json
└── vite.config.js
```

**Key ECharts features:**
- `dataZoom` (slider + mouse wheel) for arbitrary zoom
- On zoom: update `from`/`to`, re-query `/api/metrics`
- 60-second polling appends new data points to existing series
- Dark/light theme toggle (ECharts built-in themes)
- Multiple metrics displayed as separate panels

**Development workflow:**
```bash
cd html
npm run dev      # Vite dev server with /api/* proxied to Flask :5000
npm run build    # outputs to html/dist/
```

## Systemd Units

**scope1090-collector.service** — always running, restarts on failure
**scope1090-api.service** — always running, restarts on failure
**scope1090-persist.service** — oneshot: `cp /run/scope1090/live.db /var/lib/scope1090/archive.db`
**scope1090-persist.timer** — triggers persist.service every hour
**scope1090-restore.service** — runs once at boot: `cp archive.db → live.db` (if exists)

**Boot sequence:**
```
1. scope1090-restore.service  (cp archive.db → live.db)
2. scope1090-collector.service (start)
3. scope1090-api.service       (start, After=collector)
4. scope1090-persist.timer     (start)
```

## Web Server Configuration

### nginx

```nginx
server {
    listen 80;
    root /usr/share/scope1090/html;

    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### lighttpd

```
server.modules += ("mod_proxy", "mod_rewrite")

server.document-root = "/usr/share/scope1090/html"

$HTTP["url"] =~ "^/api/" {
    proxy.server = ("" => (("host" => "127.0.0.1", "port" => 5000)))
    proxy.forwarded = ("for" => 1)
}

url.rewrite-if-not-file = ("^/((?!api/).*)$" => "/index.html")
```
